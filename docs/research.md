# AI開発ポートフォリオサイト リサーチレポート

デザインテーマ: **"Warm Elegant Minimal"** — 温かく柔らかなフェミニンな美学、日本のエディトリアル感

## 1. 参考サイト分析

### 1-1. Brittany Chiang (brittanychiang.com)
- **URL**: https://brittanychiang.com/
- **特徴**: シングルページスクロール、Next.js + Tailwind CSS、セクション（About / Experience / Projects / Writing）で明確に構成。Inter フォント使用。コンテンツファースト。
- **参考ポイント**:
  - ミニマルなナビゲーション（アンカーリンク）
  - セクション間の十分な余白による視認性
  - プロジェクトカードの整理された配置
  - スクロール追従のナビゲーションハイライト
  - 控えめだが効果的なホバーアニメーション

### 1-2. Lee Robinson (leerob.com)
- **URL**: https://leerob.com/
- **特徴**: 極めてシンプルなシングルカラムレイアウト。ナビゲーションメニューなし、インラインリンクで誘導。ライト/ダークテーマ対応。タイポグラフィ主導の設計。
- **参考ポイント**:
  - コンテンツファーストの潔い設計思想
  - 余白の贅沢な使い方（ホワイトスペースの美学）
  - 装飾を排した文字中心の構成
  - 「書き手」としての個性を文章で伝える手法

### 1-3. Tamal Sen (tamalsen.dev)
- **URL**: https://tamalsen.dev/
- **特徴**: ミニマリズムと繊細なアニメーションの融合。シングルページでスキル→プロジェクト→コンタクトを展開。美しいタイポグラフィ階層。
- **参考ポイント**:
  - ファーストビューの印象的なキャッチコピー演出
  - フィーチャードプロジェクトを最初に配置する戦略
  - プロジェクトのスクリーンショットサイズを変えることで視覚リズムを生む
  - 洗練されたスクロールアニメーション

### 1-4. The Orpens (ピンク系エディトリアルデザイン)
- **URL**: Awwwards掲載ピンク系ポートフォリオ
- **特徴**: ソフトピンクのトーン、小文字タイポグラフィ、モノクロイメージとの組み合わせ。ロマンチックでエレガントなショーケース。
- **参考ポイント**:
  - ダスティピンクの効果的な使い方（アクセントとしての配色）
  - セリフ体と柔らかいカラーの組み合わせによるエレガンス
  - グレイン（粒子感）テクスチャの活用
  - 画像とタイポグラフィの調和

### 1-5. 日本のエディトリアル系Webデザイン (Siteinspire: Japanese + Minimal)
- **URL**: https://www.siteinspire.com/websites?categories=180+55+14+19
- **特徴**: 日本の「間（ma）」の美学を体現するミニマルデザイン。禅的な余白使い、明朝体の品格、シンプルな線と形。
- **参考ポイント**:
  - 日本語明朝体（Mincho）の上品さ → Noto Serif JP で再現
  - 「余白は語る」思想 — 情報を詰め込まない勇気
  - 縦書きアクセントの可能性（section titleなどに）
  - 自然な色彩とニュートラルトーンの活用

## 2. デザインシステム定義

### カラーパレット
| 用途 | カラー | コード |
|------|--------|--------|
| ベース（背景） | ウォームホワイト | `#FAF8F5` |
| サブ背景 | クリーム | `#F5F0EB` |
| メインテキスト | チャコール | `#2D2D2D` |
| サブテキスト | ウォームグレー | `#6B6B6B` |
| アクセント | ダスティピンク | `#E8A4B8` |
| アクセント（ホバー） | ディープピンク | `#D4899E` |
| ライン/ボーダー | ライトグレー | `#E8E4E0` |

### タイポグラフィ
| 用途 | フォント | ウェイト |
|------|----------|----------|
| 見出し（日本語） | Noto Serif JP | 500 / 700 |
| 見出し（英語） | Cormorant Garamond | 400 / 600 |
| 本文（日本語） | Noto Sans JP | 300 / 400 |
| 本文（英語） | Inter | 300 / 400 |
| アクセント | Caveat（手書き風） | 400 |

### スペーシング
- セクション間: `120px` (desktop) / `80px` (mobile)
- コンテンツ内余白: `64px` (desktop) / `32px` (mobile)
- 最大コンテンツ幅: `1200px`

## 3. 推奨サイト構造

### 3-1. ナビゲーション

**レイアウト**: スティッキーヘッダー、上部固定
**構成要素**:
- 左: ロゴ/名前（Cormorant Garamond, 細めのセリフ体）
- 右: メニューリンク（About / Works / Skills / Contact）
- モバイル: ハンバーガーメニュー

**デザイン詳細**:
- 背景: 透明 → スクロール時に `rgba(250, 248, 245, 0.95)` + `backdrop-filter: blur(10px)`
- フォントサイズ: 14px、レターSpacing: 0.1em
- メニューリンクのホバー: ダスティピンクのアンダーライン（左から右へ展開アニメーション）
- height: 72px (desktop) / 56px (mobile)

**アニメーション**:
- スクロールダウンで非表示、スクロールアップで再表示（hide/show on scroll direction）
- 表示/非表示は `transform: translateY()` + `transition: 0.3s ease`

---

### 3-2. ヒーローセクション

**レイアウト**: フルスクリーン（100vh）、中央揃え
**構成要素**:
- キャッチコピー（メイン）: 「AIで、未来をデザインする。」
- サブキャッチ（英語）: "Designing the Future with AI"（Cormorant Garamond, italic）
- 簡潔な説明文（1-2行）: 「AI開発エンジニアとして、最先端の技術で新しい体験を創り出しています。」
- CTAボタン: 「実績を見る」→ Worksセクションへスクロール
- 背景: AIが生成したアブストラクトなグラデーション画像 or ソフトフォーカスの幾何学パターン

**デザイン詳細**:
- キャッチコピー: Noto Serif JP, 48px (desktop) / 32px (mobile), font-weight: 700
- サブキャッチ: Cormorant Garamond, 24px, italic, ダスティピンク色
- CTAボタン: 枠線スタイル（border: 1px solid #2D2D2D）、ホバーで背景塗りつぶし
- 背景画像: `object-fit: cover` + ソフトオーバーレイ（`rgba(250, 248, 245, 0.85)`）

**アニメーション**:
- テキスト: フェードイン + 下から上へスライド（stagger: 0.2s 間隔）
- 背景: ゆっくりズームイン（scale 1.0 → 1.05, 20秒かけて）
- スクロールインジケーター: 下部に細い線 + 上下アニメーション

---

### 3-3. 自己紹介（About）セクション

**レイアウト**: 2カラム（左: プロフィール画像、右: テキスト）、モバイルは縦積み
**構成要素**:
- セクションタイトル: "About"（Cormorant Garamond, 小さめ + 下線アクセント）
- プロフィール画像: 丸型 or ソフトな角丸（border-radius: 20px）
- 名前 + 肩書き
- 自己紹介文（3-4段落）
- スキルサマリー: タグ形式（例: Python / TensorFlow / LLM / Computer Vision）

**デザイン詳細**:
- プロフィール画像枠: 薄いダスティピンクのシャドウ or ボーダー
- スキルタグ: `background: #F5F0EB`, `border: 1px solid #E8E4E0`, `border-radius: 20px`, `padding: 6px 16px`
- セクションタイトルのアンダーライン: ダスティピンクの手書き風ライン（Caveatフォントでの装飾 or SVG）
- 行間: 2.0 (日本語テキストの読みやすさ確保)

**アニメーション**:
- 画像: 左からスライドイン + フェードイン
- テキスト: 右からスライドイン + フェードイン（stagger）
- スキルタグ: 1つずつフェードイン（stagger: 0.05s）
- トリガー: Intersection Observer（画面に50%入った時点で発火）

### 3-4. 実績ギャラリー（Works）セクション

**レイアウト**: 3×2 グリッド（desktop）、2×3 グリッド（tablet）、1カラム（mobile）
**構成要素**:
- セクションタイトル: "Works"
- セクション説明: 「AI開発で携わったプロジェクトの実績です。」
- 6枚のプロジェクトカード（各カードにスクリーンショット画像）

**カード構成**:
```
┌─────────────────────┐
│                     │
│   スクリーンショット   │
│   (16:9 比率)       │
│                     │
├─────────────────────┤
│ プロジェクト名        │
│ 説明文（1-2行）      │
│ タグ: Python, LLM   │
└─────────────────────┘
```

**デザイン詳細**:
- カード: `background: #FFFFFF`, `border-radius: 12px`, `box-shadow: 0 2px 20px rgba(0,0,0,0.04)`
- 画像: `border-radius: 12px 12px 0 0`, `aspect-ratio: 16/9`, `object-fit: cover`
- カード間ギャップ: 32px (desktop) / 20px (mobile)
- ホバーエフェクト: カード全体が少し浮く（`transform: translateY(-4px)`）+ シャドウ強調
- プロジェクト名: Noto Serif JP, 18px, font-weight: 500
- 説明文: Noto Sans JP, 14px, color: #6B6B6B
- テクノロジータグ: 小さめ（12px）、ダスティピンク背景のピル型

**アニメーション**:
- カード: フェードイン + 下からスライド（stagger: 0.1s 間隔で6枚が順に表示）
- ホバー: `transition: transform 0.3s ease, box-shadow 0.3s ease`
- 画像ホバー: 軽いスケール（1.02）

### 3-5. スキル/ツール（Skills）セクション

**レイアウト**: フル幅セクション、内部は3カラムグリッド
**構成要素**:
- セクションタイトル: "Skills & Tools"
- 3つのカテゴリカード:

**カテゴリ1: AI/ML開発**
- Python, TensorFlow, PyTorch
- Scikit-learn, Hugging Face
- LLM (GPT, Claude, Gemini)

**カテゴリ2: 開発ツール**
- Git/GitHub, Docker
- AWS / GCP
- CI/CD, MLOps

**カテゴリ3: フロントエンド/その他**
- React, Next.js, TypeScript
- Figma, UI/UX Design
- データ可視化

**デザイン詳細**:
- 背景: `#F5F0EB`（クリーム系でセクションを差別化）
- カテゴリカード: 背景白、パディング48px、`border-radius: 16px`
- カテゴリタイトル: Noto Serif JP, 20px, ダスティピンク色
- スキルアイテム: リスト形式、左にドット（ダスティピンク）、右にプログレスバー or なし（シンプルに）
- アイコン: 各カテゴリにシンプルなラインアイコン

**アニメーション**:
- 3カードが左から右へ順にフェードイン（stagger: 0.15s）
- プログレスバー使用時: 左から右へバーが伸びるアニメーション

---

### 3-6. コンタクト（Contact）セクション

**レイアウト**: 中央揃え、シンプルな構成
**構成要素**:
- セクションタイトル: "Contact"
- 誘導テキスト: 「お仕事のご依頼・ご相談はお気軽にどうぞ。」
- メールリンク: `mailto:` リンク（大きめのタイポグラフィで）
- SNSリンク: GitHub / X (Twitter) / LinkedIn（アイコン横並び）

**デザイン詳細**:
- メールアドレス: Cormorant Garamond, 28px, ホバーでダスティピンク色に変化
- SNSアイコン: 24px、ラインスタイル、ホバーでダスティピンク塗りつぶし
- セクション全体を`text-align: center`で中央揃え
- 装飾: メールの上下に細いダスティピンクのライン

**アニメーション**:
- テキスト: フェードイン（中央から）
- SNSアイコン: 1つずつ下からポップイン（stagger: 0.1s）

### 3-7. フッター

**レイアウト**: フル幅、最小限の構成
**構成要素**:
- コピーライト表記: `© 2026 [Name]. All rights reserved.`
- 「Back to Top」リンク（↑ アイコン）
- サブリンク: Privacy Policy（オプション）

**デザイン詳細**:

- パディング: `40px 0`
- 「Back to Top」: ホバーでダスティピンクに変化、スムーススクロールで先頭へ

## 5. パフォーマンス最適化

- 画像: WebP形式、`loading="lazy"`
- フォント: `font-display: swap`、preconnect to Google Fonts
- CSS: カスタムプロパティで一元管理
- JS: Intersection Observer で遅延アニメーション
- 全体: Lighthouse スコア 90+ を目標

## 6. 技術スタック（推奨）

- **HTML5**: セマンティックマークアップ
- **CSS3**: カスタムプロパティ、CSS Grid、Flexbox
- **JavaScript**: Vanilla JS（軽量化優先）、Intersection Observer API
- **ホスティング**: GitHub Pages（静的サイト）
- **フォント**: Google Fonts（Noto Serif JP, Noto Sans JP, Cormorant Garamond, Inter）
