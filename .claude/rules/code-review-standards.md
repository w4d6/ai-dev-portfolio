# Code Review Standards

## 全般的なレビュー基準

### 重要度レベル

| レベル | 意味 | 対応 |
|--------|------|------|
| ❌ Critical | セキュリティ、アクセシビリティ、重大なバグ | 即座に修正必須 |
| ⚠️ Warning | パフォーマンス、保守性、ベストプラクティス違反 | 次回リリースまでに対応推奨 |
| ✅ Pass | 基準を満たしている | 対応不要 |

### 共通チェック項目

1. **セキュリティ**
   - XSS脆弱性がないこと（innerHTML、eval等の不使用）
   - 外部リンクに `rel="noopener noreferrer"` 付与
   - 機密情報（APIキー、パスワード等）がコードに含まれないこと

2. **パフォーマンス**
   - 画像のlazy loading設定
   - CSSアニメーションは `transform`/`opacity` のみ使用
   - 不要なリフロー/リペイントの回避

3. **保守性**
   - 一貫した命名規則
   - マジックナンバーの排除（定数化/CSS変数化）
   - コメントは「なぜ」を説明（「何を」ではない）

4. **互換性**
   - `prefers-reduced-motion` 対応
   - ベンダープレフィックス（必要な場合）
   - `font-display: swap` 使用
